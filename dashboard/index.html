2

Crie uma cobrança Pix pelo bot

Quando um cliente quiser comprar, seu bot faz uma chamada à API para gerar o Pix. Guarde o correlationID retornado — você vai precisar dele para saber quem pagou.

Python — criar cobrança 
# pip install requests
import requests

API_KEY = "bst_live_sua_chave_aqui"  # variável de ambiente
BASE_URL = "https://sua-api.railway.app"

# Mapeamento correlationID → user do Telegram
pedidos_pendentes = {}

def criar_cobranca(valor_centavos, telegram_user_id):
    resp = requests.post(
        f"{BASE_URL}/api/v1/charges",
        headers={
            "Authorization": API_KEY,  # sem "Bearer"
            "Content-Type": "application/json",
        },
        json={
            "value": valor_centavos,  # em centavos — R$ 29,90 = 2990
            "expiresIn": 3600,        # expira em 1 hora
        }
    )
    cobranca = resp.json()

    # Salva para depois saber quem pagou
    pedidos_pendentes[cobranca["correlationID"]] = telegram_user_id

    return cobranca
Resposta da API JSON 
{
  "correlationID": "3f8a1b2c-4d5e-6f7a-8b9c-0d1e2f3a4b5c",
  "brCode": "00020126580014BR.GOV.BCB.PIX...",  ← PIX copia e cola
  "qrCodeImage": "https://cdn.woovi.com/qr/...",     ← imagem do QR code
  "value": 2990,
  "expiresAt": "2025-01-15T15:00:00Z"
}
3

Envie o Pix para o cliente no Telegram

Use o brCode (copia e cola) e o qrCodeImage para exibir o pagamento pro cliente dentro do Telegram. Ele pode pagar direto pelo app do banco.

Python — enviar ao cliente 
# python-telegram-bot v20+
async def cmd_comprar(update, context):
    user_id = update.effective_user.id

    # Cria a cobrança
    cobranca = criar_cobranca(2990, user_id)  # R$ 29,90

    # Mensagem com o código PIX
    await update.message.reply_text(
        f"💳 *Pague R$ 29,90 via PIX*\n\nCopie o código abaixo e cole no seu banco:\n\n"
        f"`{cobranca['brCode']}`",
        parse_mode="Markdown"
    )

    # Imagem do QR code (opcional, mas ajuda muito)
    await update.message.reply_photo(
        photo=cobranca["qrCodeImage"],
        caption="📱 Ou escaneie o QR Code pelo app do banco"
    )

    await update.message.reply_text(
        "⏳ Aguardando confirmação do pagamento..."
    )
4

Configure o Webhook no painel

Quando o cliente pagar, o BISTECA precisa avisar seu bot. Para isso, você configura uma URL pública do seu servidor que vai receber a notificação de pagamento.

No painel: Configurações → Webhook URL
https://meu-bot.railway.app/webhook/pix

# Regras:
# ✅ Precisa ser HTTPS e acessível pela internet
# ✅ Deve responder com HTTP 200 em até 5 segundos
# ✅ O BISTECA retentará automaticamente se falhar
Desenvolvendo localmente? Use ngrok para expor seu servidor: ngrok http 3000 e cole a URL gerada.
5

Receba a confirmação e libere o acesso

Quando o pagamento for confirmado, o BISTECA faz um POST na sua URL com os dados da cobrança. Seu bot recebe o correlationID, identifica o cliente e libera o acesso ao grupo VIP.

Payload recebido no webhook JSON 
{
  "event": "charge.paid",
  "data": {
    "correlationID": "3f8a1b2c-4d5e-6f7a-8b9c-0d1e2f3a4b5c",
    "value": 2990,
    "paidAt": "2025-01-15T14:32:10.000Z"
  },
  "timestamp": "2025-01-15T14:32:10.123Z"
}
Python + Flask — endpoint de webhook 
# pip install flask
from flask import Flask, request, jsonify
import asyncio

app = Flask(__name__)

# Link entre correlationID e Telegram user_id
# (pedidos_pendentes é compartilhado com o código do bot)

@app.route("/webhook/pix", methods=["POST"])
def webhook_pix():
    payload = request.json

    if payload.get("event") == "charge.paid":
        correlation_id = payload["data"]["correlationID"]

        # Quem pagou?
        user_id = pedidos_pendentes.pop(correlation_id, None)

        if user_id:
            # Adiciona ao grupo VIP do Telegram
            asyncio.run(bot.approve_chat_join_request(GRUPO_VIP_ID, user_id))

            # Avisa o cliente
            asyncio.run(bot.send_message(
                chat_id=user_id,
                text="✅ Pagamento confirmado! Bem-vindo ao grupo VIP! 🎉"
            ))

    # SEMPRE responda 200 — senão o BISTECA vai retentar
    return jsonify({"ok": True}), 200
Pronto! O fluxo completo funciona assim: cliente pede → bot gera PIX → cliente paga → BISTECA avisa o bot → bot libera o grupo. Tudo automático.
